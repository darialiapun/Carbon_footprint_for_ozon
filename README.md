**Carbon_footprint_for_ozon** - расширение для Google Chrome, которое рассчитает углеродный след товаров на маркетплейсе Ozon

## Обзор

**Carbon_footprint_for_ozon** — это браузерное расширение для Google Chrome, которое помогает пользователям осознанно подходить к потреблению, визуализируя экологический след покупаемых товаров. Расширение в реальном времени анализирует карточку товара, вычисляет объем выбросов $\text{CO}_2$ (включая производство и упаковку) и выводит эти данные в виде виджета.

## Особенности

*   **Многоуровневый расчет эмиссии:** Интеллектуальная система определения коэффициентов выбросов на основе состава материала $\rightarrow$ категории товара $\rightarrow$ средних значений (fallback).
*   **Анализ упаковки:** Автоматический расчет веса и углеродного следа полиэтиленовой упаковки (LDPE) на основе габаритов товара.
*   **Геймификация через «Эко-Героев»:** Визуальное сравнение следа товара с жизненным циклом животных, страдающих от парниковго эффекта (пингвинов, белых медведей и др.) для более наглядного восприятия цифр. Герои созданы с помощью веб-приложения **[Emoji Maker](https://github.com/darialiapun/emogi-maker)** — интеллектуального генератора 3D-эмодзи на базе нейросети Flux и Wavespeed AI.
*   **Мониторинг месячного лимита:** Индикатор прогресса, показывающий, какой процент от среднего рекомендуемого месячного лимита человека $\text{CO}_2$ (166 кг) занимает данный товар.
*   **Бесшовная интеграция с Ozon:** Поддержка SPA-навигации (Single Page Application) и динамической подгрузки характеристик через `MutationObserver`.

## Технические компоненты

### Data Pipeline
Расширение реализует сложный процесс извлечения данных из динамического DOM:
*   **Скрапинг характеристик:** Поиск веса, габаритов и состава материала через семантические паттерны (регулярные выражения) в разделе «О товаре» и технических характеристиках.
*   **JSON-LD Parsing:** Извлечение структурированных данных из мета-тегов и скрытых скриптов страницы для повышения точности.
*   **Интеллектуальный fallback:** Если вес не указан в карточке, система определяет категорию товара (например, «обувь» или «электроника») и подставляет репрезентативный средний вес.

### Логика расчета (Emission Engine)
Расчет производится по строгому приоритету:
1.  **По материалу:** Если найден состав (например, «хлопок»), используется коэффициент конкретного материала $\times$ вес.
2.  **По категории:** Если состав неизвестен, используется коэффициент категории $\times$ вес.
3.  **По средним значениям:** Если вес не найден, используется `FALLBACK_WEIGHTS` для данной категории.

**Формула общего следа:**
$$\text{Total} = (\text{Weight} \times \text{Coefficient}) + \text{PackagingEmission}$$

### Интерфейс (UI/UX)
*   **Динамический виджет:** Инъекция в правую колонку Ozon с использованием изолированных стилей.
*   **SVG-визуализация:** Кольцевой индикатор прогресса с динамическим изменением цвета (от зеленого к красному).
*   **Стикеры-герои:** Использование `mix-blend-mode: multiply` для создания эффекта наклейки животных поверх интерфейса.

## Требования

*   Браузер Google Chrome (или любой на базе Chromium).
*   Версия Manifest V3.
*   Доступ к интернет-соединению для загрузки страниц Ozon.

## Настройка и установка

1.  Скачайте репозиторий:
    ```bash
    git https://github.com/darialiapun/Carbon_footprint_for_ozon.git
    ```
2.  Откройте браузер Chrome и перейдите по адресу: `chrome://extensions/`
3.  Включите **«Режим разработчика»** (Developer mode) в правом верхнем углу.
4.  Нажмите кнопку **«Загрузить распакованное расширение»** (Load unpacked) и выберите папку с проектом.
5.  Перейдите на сайт Ozon в любую карточку товара.

## Использование

1.  Откройте страницу любого товара на Ozon.
2.  Дождитесь автоматического появления виджета «Углеродный след» в правой части экрана (под блоком цены).
3.  Ознакомьтесь с расчетом:
    *   **Процент:** Сколько % от месячного лимита $\text{CO}_2$ занимает товар.
    *   **Сравнение:** Какой процент от «жизненного следа» животного составляет этот товар.

## Архитектура системы

Процесс работы расширения представляет собой конвейер (pipeline):

**Сбор данных** $\rightarrow$ **Нормализация** $\rightarrow$ **Расчет** $\rightarrow$ **Рендеринг**

1.  **Сбор:** `collectLabelValuePairs` $\rightarrow$ `extractFromScriptJson` $\rightarrow$ `triggerCharacteristicsReveal`.
2.  **Нормализация:** Приведение всех единиц измерения к килограммам (`parseWeightToKg`) и миллиметрам (`parseDimensionsMm`).
3.  **Расчет:** `computeComprehensiveEmission` (выбор ветки: материал $\rightarrow$ категория $\rightarrow$ fallback).
4.  **Рендеринг:** `renderWidget` (создание DOM-элементов и расчет SVG-смещений).
